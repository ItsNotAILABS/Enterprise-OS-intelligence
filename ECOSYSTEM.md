## 9 — THE DOCUMENT INTELLIGENCE PIPELINE

**Official name:** Mundator Cognitus — Cognitive Document Intelligence  
**Engine identity:** [E13] Evidence Registry + [E14] Dispute/Correction  
**Location:** `tools/doc-sanitizer.js`

### Two-Pass AI Flow

```
INCOMING DOCUMENT
       │
       ▼
[PASS 1 — E13/E14 — INGEST + FIX]
  E13: Detect — brand strings, canister IDs, API keys, logic code blocks
  E14: Auto-fix — replace all detectable sensitive content
  Write cleaned file to disk
       │
       ▼
[PASS 2 — E13 — STRICT VERIFY]
  Re-read the file PASS 1 wrote
  Confirm zero sensitive patterns survive
  Fail loudly if anything escaped PASS 1
       │
       ▼
[COMMIT — E15 — EXPORT]
  Only commit if both passes pass
  Git auto-commit with [E13·E14] signature
  GitHub Actions step summary published
```

**Modes:**
- `node tools/doc-sanitizer.js papers/` — PASS 1 (detect + fix)
- `node tools/doc-sanitizer.js papers/ --verify` — PASS 2 (strict verify, read-only)

**Workflow files:**
- `sovereign-intake.yml` — PASS 1 → PASS 2 → commit (auto-intake)
- `doc-clean.yml` — PASS 2 only (guard check on PRs)
