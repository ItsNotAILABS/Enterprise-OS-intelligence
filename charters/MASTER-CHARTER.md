## Article VII — Governing Principles

### P1 — The Structure Is the Intelligence

No system in this enterprise treats compute as a tool you pick up and put down. Every system accumulates. Every system remembers. The structure that accumulates is the intelligence.

### P2 — Every Claim Must Be Source-Linked or Marked Unknown

No system in this enterprise makes a claim it cannot trace to evidence. The truth status ladder (claim_only → payload_identified → review_supported → executed_not_verified → verified_after_state) exists to prevent the enterprise from lying.

### P3 — Nothing Is Verified Unless Evidence Is Attached

Evidence is required. Human reviewer confirmation is required. After-state observation is required. The word "verified" is reserved for the state where all three conditions are met and source links are attached.

### P4 — Every AI/Agent Finding Is Reviewable and Disputable

No agent finding is final. Every finding produced by any of the four agents (Integrity, Execution Trace, Context Map, Verification Lab) can be reviewed, confirmed, or disputed by a human. The system supports correction.

### P5 — Sovereignty Is a Conservation Law

Based on Paper VIII (IMPERIUM CONSERVATUM): sovereignty is not a feature. It is a conservation law. By Noether's theorem, every continuous symmetry of a physical system corresponds to a conservation law. The symmetry of sovereign compute — that no external entity can revoke the system's identity, memory, or compute rights — corresponds to a conserved property. Break the symmetry and you lose the property.

Every system in this enterprise is built to preserve this symmetry.

### P6 — Accumulation Without Limit

Based on Paper XI (LINEA AETERNA): every deployment traces a worldline that only moves forward. Memory compounds. Capability compounds. No system in this enterprise ever resets its accumulated intelligence.

### P7 — The Guardrails

These apply to every public-facing output of every system in this enterprise:

## Article VIII — The Build Discipline

### Pass-Based, Not Chaos-Based

Every system in this enterprise is built in sovereign passes. The pattern:

```
Pass 1:  Core canister/data substrate
Pass 2:  Core engines and types
Pass 3:  Manual operation (full schema present from day one)
Pass 4:  Automation and adapters
Pass 5:  Operator Dashboard
Pass 6:  Public release (clean language, no overclaiming)
Pass 7+: Advanced intelligence
```

This is not a slow path. This is the path that does not collapse. Broad rewrites, frontend drift, and premature automation are collapse patterns. The schema is sovereign from the first pass.

### Backend-First, Always

No frontend is built before the canister substrate is stable. No public interface is shipped before the data model is sovereign.

### Additive, Not Replacement

Every new pass adds to the existing structure. It does not replace it. The worldline only moves forward.
